# Mood → Palette Catalog

자연어 묘사를 4색 hex 팔레트로 매핑하는 카탈로그. 새로 모드(`create-flow.md`) Step 2 에서 사용. 각 무드는 (배경 / 표면 / 텍스트 / 액센트) 4색 hex 를 가진다.

> 사용법: 사용자 묘사 → 가장 가까운 카탈로그 무드 2개 + 즉석 생성 1개 = 총 3안 제시. 각 후보에 4색 hex 와 한 줄 설명을 보여준 뒤 사용자가 선택. 사용자가 "이거 베이스, 액센트만 더 따뜻하게" 처럼 미세 조정을 요청하면 그것까지 반영하여 최종 결정.

## 무드 키워드 → 후보 매핑 가이드

사용자 묘사에 다음 키워드가 있으면 우선 후보로 고려:

| 키워드 | 후보 (우선순위순) |
|---|---|
| 따뜻, 카페, 종이, 책 | warm-cafe, gruvbox-warm, sunset-warm |
| 차가운, 시원, 북유럽, nord | nordic-cold, nordic-light, tokyo-rain |
| 미니멀, 깔끔, 노션, 화이트 | notion-clean, nordic-light, brutalist-mono |
| 다크, 어두운 | nordic-cold, dracula, tokyo-rain, gruvbox-warm |
| 라이트, 밝은 | notion-clean, nordic-light, warm-cafe, sakura |
| 자연, 숲, 그린 | forest-night, gruvbox-warm |
| 코딩, 개발자, 코드 | dracula, tokyo-rain, gruvbox-warm, nordic-cold |
| 레트로, 터미널, 빈티지 | retro-terminal, gruvbox-warm |

매핑이 모호하면 라이트/다크 분기 후 각 계열에서 1개씩 골라 제시.

## 즉석 생성 가이드 (3번째 후보)

카탈로그에 없는 컨셉이면 다음 원칙으로 즉석 생성:

1. **배경**: 라이트면 `#f0~ff` 명도, 다크면 `#10~30` 명도
2. **표면**: 배경과 명도 차 5~10 (사이드바·코드블록과 본문 구분)
3. **텍스트**: 배경과 대비 비율 ≥ 7:1 (WCAG AAA)
4. **액센트**: 묘사에서 컨셉 색 추출. 채도 60~80% 권장. 채도 낮으면 음음, 너무 높으면 눈에 띔
5. 4색 모두 hex 로 명시
